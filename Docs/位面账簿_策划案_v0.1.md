# 位面账簿（The Plane Ledger）
## 完整策划案 v0.1

---

# 第一章 产品定位

## 1.1 基本信息

- 产品名称：位面账簿（The Plane Ledger）
- 发行平台：Steam（PC，.exe）
- 定价策略：买断制，6元
- 商业模式：内容完全不收费，所有装饰通过记账行为解锁，接入 Steam 成就系统与云存档

## 1.2 核心定位

一款基于 Steam 平台的独立游戏化理财工具。通过游戏化机制驱动职场人群养成记账习惯，核心体验是：**记录让位面空间有了颜色**。

## 1.3 目标用户

职场人群，25～35岁，有记账意愿但难以坚持，对轻度游戏化收集玩法有兴趣。

## 1.4 世界观

主神空间原本是一片荒芜。你的每一笔记录，让这片空间有了色彩、有了秩序、有了生命。账簿是你与位面之间的契约，思想残章是位面馈赠的智慧碎片，装饰是空间进化的印记。

# 第二章 核心功能模块

## 2.1 功能模块总览

| 模块 | 名称 | 定位 |
|------|------|------|
| 模块一 | 日常消耗矩阵 | 碎片化即时记账 |
| 模块二 | 本月统计面板 | 大额低频与收入管理 |
| 模块三 | 年度平摊中心 | 周期性支出拆解 |

设计理念：按**消耗属性**而非商品类目拆分，用户只需问自己一个问题：这笔钱是今天即时消耗的，还是周期性的固定支出？

---

## 2.2 日常消耗矩阵（Daily Matrix）

**定位**：纯碎片的、即时性的、当天不记就忘的支出。

**交互**：单页大表格，不支持弹窗。回车/Tab/方向键自动跳转下一行，极低摩擦。

**分类体系**：

| 大类 | 子类 |
|------|------|
| 能量摄入（饮食） | 早餐、午餐、晚餐、零食/饮料/下午茶 |
| 空间位移（短途出行） | 公交/地铁/单车、打车/网约车 |
| 即时物欲（小额随机） | 线上购物、线下购物 |
| 习惯消耗（嗜好品） | 烟/酒/咖啡/电子烟弹等 |
| 随机娱乐 | 电影票/门票/单次游戏氪金、理发/护理/按摩等 |

---

## 2.3 本月统计面板（Monthly Overview）

**定位**：周期性的、大额的、通常有账单支撑或固定扣费的支出与收入。

**收入管理**：
- 劳动收入（工资/兼职）
- 资产收益（利息/分红/租金/过往作品收益）
- 投资收益（股票/基金/黄金）

**非日付支出**：
- 房租/房贷、车贷、物业取暖费
- 燃油、保险、分期还款
- 水电网费、保洁

**核心指标**：
- 生存天数：当前存款 ÷ 日均支出（直观展示财务安全感）
- 被动收入占比：进度条显示，目标是覆盖100%支出

---

## 2.4 年度平摊中心（Amortization Center）

**操作逻辑**：用户一次性录入项目名称、总金额、覆盖月份，系统自动在对应月份的统计中生成虚拟「位面折旧费」。

**适用类型**：停车费、物业费、取暖费、保险等年付/季付项目。

**示例**：录入「冬季取暖费 2400元 / 12个月」→ 每月自动计入200元折旧费。

# 第四章 核心循环

## 4.1 三层循环结构

**日循环（每天）**
```
打开游戏 → 触发登录奖励（装饰×1）
→ 进入主界面
→ 点击账本记账
→ 点击结算
→ 触发结算奖励（残章×1 + 代币×10）+ 成就检测
→ 查看掉落 → 换装主界面
```

**周/月循环**
```
连续记账累积 → 触发里程碑成就
→ 获得指定稀有/史诗装饰 + 大额代币
→ 主界面持续进化
```

**长期循环**
```
完成大型成就 → 解锁传说装饰（唯一来源）
→ 驱动长期留存
```

## 4.2 成就检测时机

| 检测时机 | 触发节点 | 适用类型 |
|----------|---------|---------|
| 实时检测 | 操作完成时 | 首次操作类、累计数量类 |
| 结算时检测 | 点击结算按钮 | 当日行为类、连续天数类 |
| 月末检测 | 每月最后一天结算 | 月度财务类 |

# 第五章 数值体系

## 5.1 代币获取规则

| 来源 | 代币数量 | 说明 |
|------|---------|------|
| 每日记账结算 | +10枚 | 每日限1次 |
| 小型成就 | +10～30枚 | 登录、首次操作类 |
| 中型成就 | +50～100枚 | 习惯养成、预算管理类 |
| 大型成就 | +150～300枚 | 长期坚持、财务目标类 |
| 传说成就 | +300～500枚 | 隐藏成就、百日以上 |

**注：无每笔记账给代币机制，防止刷单。连续里程碑即成就，不单独列出。**

## 5.2 代币消耗规则

| 消耗项目 | 价格 |
|----------|------|
| 指定购买残章 | 30枚/张 |
| 指定购买普通装饰 | 50枚/件 |
| 指定购买稀有装饰 | 150枚/件 |
| 重置当日随机掉落池 | 20枚/次 |

史诗/传说装饰不可代币购买，仅通过成就获得。

## 5.3 掉落体系

### 每日登录掉落
- 触发条件：当天首次打开游戏
- 掉落内容：随机装饰 ×1
- 特殊规则：连续登录第7天，掉落等级提升一档

### 每日记账结算掉落
- 触发条件：当天完成至少1笔记账后点结算
- 掉落内容：随机残章 ×1 + 代币 ×10
- 每日上限：1次

**结论：每天登录+记账，最多掉落 装饰×1 + 残章×1 + 代币×10。**

### 装饰掉落稀有度权重

| 稀有度 | 权重 | 可进随机池 |
|--------|------|-----------|
| 普通 | 70% | 是 |
| 稀有 | 25% | 是 |
| 史诗 | 5% | 是 |
| 传说 | 0% | 否，仅成就获得 |

### 保底机制
- 装饰：连续3天未掉稀有以上，第4天保底稀有
- 残章：连续4次未掉稀有以上，第5次保底稀有
- 无传说保底

### 重复获取处理
- 普通件重复：转化10枚代币
- 稀有件重复：转化20枚代币
- 史诗件重复：转化50枚代币

## 5.4 月均节奏估算

| 指标 | 数值 |
|------|------|
| 每日最多掉落 | 装饰×1 + 残章×1 |
| 月均装饰掉落（每天登录） | 约30件 |
| 月均残章掉落（每天记账） | 约30张 |
| 月均史诗装饰期望 | 约1.5件 |
| 全年成就代币总供给 | 约14230枚 |
| 全年代币消耗估算 | 约5000枚 |

## 5.5 彩蛋奖励（不计入成就系统）

| 彩蛋名 | 触发条件 | 奖励 |
|--------|---------|------|
| 位面自由宣言 | 被动收入占比首次达到100% | 专属动态壁纸「自由宣言」+ 2000代币 + 专属解锁动画 |

# 第八章 装饰品系统

## 8.1 槽位与风格体系

7个独立换装槽位，6种基础风格，理论组合数 6^7 = 279,936 种。

**风格体系**：

| 风格 | 视觉关键词 |
|------|-----------|
| 默认 | 简洁、中性 |
| 克苏鲁 | 触手、深海、眼球、孢子 |
| 古风 | 云纹、锦缎、宣纸、水墨 |
| 现代 | 简约、几何、工业感 |
| 未来 | 全息、数据流、悬浮、光纤 |
| 奇幻 | 魔法阵、星图、水晶、精灵 |

## 8.2 稀有度规则

| 稀有度 | 日常掉落权重 | 代币购买 | 主要来源 |
|--------|------------|---------|---------|
| 普通 | 70% | 50枚 | 日常掉落 |
| 稀有 | 25% | 150枚 | 日常掉落 / 成就 |
| 史诗 | 5% | 不可购买 | 成就绑定为主 |
| 传说 | 0% | 不可购买 | 仅成就/里程碑 |

## 8.3 首发内容目标

- 账本槽位：首发30件
- 其余6个槽位：每槽位6件（覆盖全部风格）
- **首发总目标：约72件**

## 8.4 扩充规则

新增装饰时按以下模板填写：

```
ID / 名称 / 槽位 / 风格 / 稀有度 / 含特效 / 获取方式 / 解锁描述 / 视觉描述 / 版本
```

传说件条件：有专属解锁动画 + 与重要成就强绑定 + 全服唯一来源。
