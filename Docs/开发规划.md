# 位面账簿（The Plane Ledger）Unity 开发规划

## Context

基于三份策划文档（策划案v0.1、装饰品系统规范v0.1、残章卡片列表v0.1），从零搭建一款 Steam 平台的游戏化记账工具。当前 `D:\ClaudePlan` 目录为空，无既有代码。用户本机已有 Unity 2021.3.8f1。

**核心目标**：通过游戏化机制（装饰收集、卡牌掉落、成就系统）驱动用户养成记账习惯。买断制6元，无内购。

---

## 一、技术选型

| 决策项 | 选择 | 理由 |
|--------|------|------|
| Unity版本 | 2021.3.x LTS | 用户已安装，稳定 |
| 渲染管线 | 内置渲染管线（Built-in） | 2D项目，无需URP/HDRP |
| UI框架 | **uGUI**（非UI Toolkit） | 2021.3的UI Toolkit运行时不完善；uGUI的InputField更适合高频文本输入（日常矩阵） |
| 数据存储 | **JSON + 文件系统** | 数据量小，零依赖，完美兼容Steam Auto-Cloud |
| JSON库 | Newtonsoft.Json (Json.NET) | 支持Dictionary序列化，比JsonUtility灵活 |
| Steam集成 | Steamworks.NET | 成就 + Auto-Cloud（后台配置路径即可，无需代码处理云存档） |
| 字体 | TextMeshPro + 思源黑体/Noto Sans CJK | 中文支持，Dynamic SDF模式 |
| 架构模式 | 服务定位器 + 事件总线 | 项目规模适中，不需要Zenject等DI框架 |

## 三、核心架构设计

### 3.1 命名空间

`PlaneLedger.Core` / `PlaneLedger.Data` / `PlaneLedger.Accounting` / `PlaneLedger.PlaneSpace` / `PlaneLedger.Collection` / `PlaneLedger.Achievement` / `PlaneLedger.Currency` / `PlaneLedger.Steam` / `PlaneLedger.UI`

### 3.2 关键Manager/Service

| 类名 | 职责 |
|------|------|
| **GameManager** | 生命周期管理，Boot启动流程，日结算触发，退出自动保存 |
| **SaveSystem** | JSON序列化/反序列化，自动备份，版本迁移链 |
| **EventBus** | 模块间解耦通信（Subscribe/Publish） |
| **TokenManager** | 代币余额管理（Earn/TrySpend） |
| **DailyMatrixService** | 日常条目CRUD，按日期索引，日均消费计算 |
| **MonthlyOverviewService** | 收入管理，生存天数计算，被动收入比 |
| **AmortizationService** | 平摊项管理，自动月度分摊 |
| **DecorationSystem** | 装饰解锁/装备/查询，7槽位独立管理 |
| **WisdomCardSystem** | 卡牌解锁/查询/按阵营筛选 |
| **DropSystem** | 掉落概率+保底机制+重复转Token |
| **AchievementManager** | 解锁状态管理，奖励发放 |
| **AchievementTracker** | 监听EventBus事件，自动判定164个成就条件 |
| **SteamAchievementBridge** | 游戏成就→Steam成就映射同步 |

### 3.3 核心数据模型

- **SaveData**：Version, PlayerProfile, AccountingData, CollectionData, AchievementProgressData, SettlementData
- **DailyEntry**：Date, Category, SubCategory, Amount, Note
- **CollectionData**：UnlockedDecorationIds, UnlockedCardIds, EquippedDecorations(slot→id字典), PityCounters
- **AchievementProgressData**：Progress字典(id→进度值), UnlockedIds集合, ConsecutiveStreaks

### 3.4 ScriptableObject（静态配置）

- **DecorationSO**：Id, Name, Icon, Sprite, Slot, Style, Rarity, TokenPrice, UnlockAchievementId
- **WisdomCardSO**：Id, Name, BookSource, Faction, Rarity, OriginalQuote, PlaneAnnotation
- **AchievementSO**：Id, SteamApiName, Name, Description, Category, TargetValue, IsHidden, Rewards[]

### 3.5 事件定义

记账：`DailyEntryAdded`, `DailySettlementCompleted`, `MonthlyIncomeUpdated`
收集：`DecorationUnlocked`, `DecorationEquipped`, `WisdomCardUnlocked`
成就：`AchievementUnlocked`, `AchievementProgressUpdated`
货币：`TokensEarned`, `TokensSpent`
系统：`DayChanged`, `SaveRequested`

## 六、关键技术要点

### 日常消耗矩阵（核心体验）
- 用 `TMP_InputField` + 自定义 `OnSubmit/OnTab` 处理
- Enter→下一行同类别，Tab→右侧下一类别
- 数据结构：`Dictionary<DateTime, List<DailyEntry>>`（非二维数组）
- 行组件使用对象池

### 保底系统
- 装饰：连续3次无Rare+→第4次保底Rare
- 卡牌：连续4次无Rare+→第5次保底Rare
- pityCounter必须持久化到存档

### 存档安全
- 每次Save前自动备份上一份
- 启动时校验JSON完整性，损坏→回退备份
- 简单AES加密（防普通用户直接改文件）
- Steam Cloud作为额外保底

### 跨日处理
- 每次应用获得焦点时检查 `LastSettlementDate < Today`
- 用户多天不打开→正确断开连续登录，不补发中间天数掉落
